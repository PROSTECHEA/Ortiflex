5.  Diseño  de  la  Seguridad  
El  sistema  implementará  múltiples  estrategias  para  garantizar  la  seguridad  de  los  datos:  
●  Autenticación  basada  en  tokens  de  sesión  (Flask-Login  o  JWT)  
 
6  

●  Hash  de  contraseñas  con  bcrypt  para  evitar  filtraciones  
 ●  Validaciones  en  frontend  (JS)  y  backend  (Python)  para  prevenir  entradas  maliciosas  
 ●  Prevención  de  ataques  XSS  y  CSRF  con  Flask-WTF  
 ●  Control  de  accesos  basado  en  roles  (admin,  soporte,  usuario)

nuevas

 ●  Backend  modularizado  con  blueprints  de  Flask  
 ●  Posible  integración  futura  con  servicios  REST  externos  
 ●  Despliegue  mediante  servidores  Apache/Nginx  con  WSGI  
 ●  Opcional:  contenedorización  con  Docker  para  ambientes  de  desarrollo  y  producción  
 ●  Respaldo  automático  de  la  base  de  datos  mediante  scripts  programados

●  Descripción :  Paneles  en  tiempo  real  para  monitorear  existencias  y  destinos.
